svh: Small Viewport Height

Visible viewport when browser chrome is collapsed

In Simple word: It measures the height of the part of the screen that is actually visible right now, even after the browser’s top and bottom bars shrink or hide.

Use case: Avoid content being hidden behind the URL bar or bottom nav

Bottom text

lvh: Large Viewport Height

Maximum possible screen height the device can show including top, bottom tool bars

It pretends browser’s top bar (URL bar) and bottom bar are not exist and it takes the full screen no matter if the browser bars shrink or not.

Use case: When you want full screen layout even if URL bar shrinks on scroll, not realiable for mobile

Bottom text

dvh: Dynamic Viewport Height

100% of the dynamic viewport height if 100dvh, adjusting when UI shows/hides (like URL bars on mobile).

Use case: Very useful on mobile Safari / Chrome. Works in browsers that support dvh.

Bottom text

vh: Viewport Height

vh is height decided by browser: height based on how the browser currently defines the viewport
it includes the space taken by the browser’s top and bottom bars, which may shrink or hide when scrolling.

Use case: When you want full screen layout even if URL bar shrinks on scroll, not realiable for mobile
